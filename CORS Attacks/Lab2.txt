// Whitelisted Null Origin Vulnerability
What it is: Servers that trust Origin: null can be exploited by attackers to bypass CORS restrictions.

// When browsers send Origin: null:

1. Cross-origin redirects
2. Requests from serialized data
3. Requests using file: protocol
4. Sandboxed cross-origin requests (<iframe sandbox>)


// How the attack works:

1. Server whitelists null origin (often for local dev purposes)

2. Server responds with Access-Control-Allow-Origin: null + Access-Control-Allow-Credentials: true

3. Attacker crafts a sandboxed <iframe> — which naturally sends Origin: null.

4. Request passes the whitelist check.

5. Attacker gains cross-domain access to sensitive data.


The vulnerable response pattern:

Access-Control-Allow-Origin: null
Access-Control-Allow-Credentials: true

Fix: Never whitelist null as a trusted origin in production — use explicit, validated origin values instead.
