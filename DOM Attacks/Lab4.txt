// Lab: DOM-based open redirection

This lab contains a DOM-based open-redirection vulnerability. To solve this lab, exploit this vulnerability and redirect the victim to the exploit server.



// Solution

1. The blog post page contains the following link, which returns to the home page of the blog:

<a href='#' onclick='returnURL' = /url=https?:\/\/.+)/.exec(location); if(returnUrl)location.href = returnUrl[1];else location.href = "/"'>Back to Blog</a>

The url parameter contains an open redirection vulnerability that allows you to change where the "Back to Blog" link takes the user. 


2. Inject the following URL Followed by the exploit server:

https://0a29002603ede36480a5711b00c50032.web-security-academy.net/post?postId=6&url=https://exploit-0a2d00020350e31480ef7048018600d1.exploit-server.net/exploit

This URL parameter redirect the victim to external website.
